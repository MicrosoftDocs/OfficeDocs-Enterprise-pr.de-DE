Weitere Informationen finden Sie unter [upgrade von Office 2007-Servern und Clients zu erleichtern](upgrade-from-office-2007-servers-and-products.md).
  
## <a name="what-does-end-of-support-mean"></a>Welche funktioniert Ende Mittelwert Support?

Microsoft-Produkten wie PerformancePoint Server 2007 SP3, ProClarity-Software und SharePoint Server 2007 SP3 hatte eine Supportlebenszyklus während der Microsoft neue Features, Updates und Sicherheitsupdates bietet. Des Lebenszyklus für ein Produkt in der Regel dauert zehn Jahren ab dem Zeitpunkt der ersten Version des Produkts, und das Ende dieses Lebenszyklus wird als das Produkt Ablauf des Supports bezeichnet. Wie ProClarity, PerformancePoint Server und SharePoint Server 2007 deren Ablauf des Supports erreicht haben, wird Microsoft nicht mehr zur Verfügung:
  
- Technischen Support für Probleme, die auftreten können
    
- Fehler korrigiert für Probleme, die ermittelt werden und kann, die die Stabilität und Verwendbarkeit von Servern auswirken
    
- Sicherheitsupdates für Sicherheitsrisiken, ermittelt werden und möglicherweise sorgen Servern oder Anwendungen anfällig für Sicherheitslücken
    
- Zeitzonenupdates
    
Die Installation von ProClarity, SharePoint Server 2007 SP3 und PerformancePoint Server 2007 SP3 wird weiterhin ausgeführt, obwohl Unterstützung beendet wurde. Jedoch wird dringend empfohlen, dass Sie diese Anwendungen so bald wie möglich migrieren.

### <a name="use-power-bi-in-the-cloud-or-on-premises"></a>Verwenden von Power BI (in der Cloud oder lokale)

Power BI ist eine Sammlung von Business Analytics Tools zum Analysieren von Daten und Freigeben von Insights. Mit Power BI können Sie interaktive Berichte und Dashboards mithilfe von lokaler oder online-Datenquellen erstellen. Personen können anzeigen und verwenden, die Berichte und Dashboards mithilfe von ihren Computern oder mobilen Geräten.
  
Power BI ist nicht in Office 365 oder SharePoint Server enthalten, aber eine Separate, anbietet, dass Power BI Desktop, Power BI-Gateways und der Power BI-Dienst enthält. Power BI integriert auch mit SharePoint Online. Sie können mit Power BI kostenlos und abhängig von Ihren Daten Verwendung und Business Anforderungen, mit Power BI Pro Upgrade beginnen. Weitere Informationen finden Sie unter [Was ist Power BI?](https://go.microsoft.com/fwlink/?linkid=841341)
  
### <a name="use-reporting-services-on-premises"></a>Verwenden von Reporting Services (lokal)

SQL Server Reporting Services bietet eine robuste berichterstellungslösung zusammen mit der Möglichkeit zum Installieren und Konfigurieren von Reporting Services im einheitlichen Modus oder integrierten SharePoint-Modus. Sie können mithilfe von mehrere Tools, einschließlich Berichts-Designer, Berichts-Generator und Power View Berichte erstellen. Mit der neuesten Version von SQL Server können Sie auch die SQL Server Mobile Bericht Publisher verwenden, zum Übermitteln von Berichte, die Skalierung an eine beliebige Bildschirmgröße, wenn Sie für Ihre Organisation die Möglichkeit, Berichte auf ihren mobilen Geräten nutzen. Weitere Informationen finden Sie unter [SQL Server Reporting Services (SSRS): erstellen, bereitstellen und Verwalten von Berichten für mobile und Seitennummerierung](https://go.microsoft.com/fwlink/?linkid=841342).
  
### <a name="use-performancepoint-services-on-premises"></a>Verwenden von PerformancePoint Services (lokal)

Wie Sie wissen, wurde PerformancePoint Server 2007 mit SharePoint Server 2007 separat erworben. Beginnend mit SharePoint Server 2010 ist die PerformancePoint-Dienste einer dienstanwendung in SharePoint Server. Dies bedeutet, dass Sie keine separate Serverlizenzen oder Hardware erwerben, um PerformancePoint-Dienste zu verwenden.
  
Zum Verschieben von PerformancePoint Server 2007 auf PerformancePoint Services in einer neueren Version von SharePoint Server verschieben, und Konfigurieren von PerformancePoint Services. Die Version von SharePoint Server, die Sie verschieben, bestimmt, ob Sie Ihre vorhandenen Dashboard-Inhalte von PerformancePoint Server 2007 auf PerformancePoint Services importieren können.
  
- Wenn Sie ein Upgrade auf SharePoint Server 2010 sind, können Sie Ihre PerformancePoint Dashboard-Inhalte auf PerformancePoint Services in SharePoint Server 2010 von PerformancePoint Server 2007 importieren. Erfahren Sie mehr darüber, wie dies funktioniert, finden Sie unter [Import-Assistent: PerformancePoint Server 2007-Inhalte auf SharePoint Server 2010](https://go.microsoft.com/fwlink/?linkid=838873).
    
- Wenn Sie auf SharePoint Server 2013 oder SharePoint Server 2016 verschieben, müssen Sie wahrscheinlich neue Dashboardinhalte (Datenquellen, Berichte, Scorecards und Dashboardseiten) zu erstellen.
    
Um auf Ihre Upgradeplan PerformancePoint Services beginnen, finden Sie in den folgenden Ressourcen:
  
1. [SharePoint Server 2007-Ablauf des Supports Roadmap](sharepoint-2007-end-of-support.md)
    
2. Wenn Sie, welche Version von SharePoint wissen können Sie auf den entsprechenden Artikel für PerformancePoint Services finden Sie unter verschieben:
    
  - [Planen für PerformancePoint Services (SharePoint Server 2010)](https://go.microsoft.com/fwlink/?linkid=841363)
    
  - [Übersicht über PerformancePoint-Dienste in SharePoint Server 2013](https://go.microsoft.com/fwlink/?linkid=841551)
    
  - [Übersicht über PerformancePoint-Dienste in SharePoint Server 2016](https://go.microsoft.com/fwlink/?linkid=874704)
    
Wenn Sie ein auf PerformancePoint-Dienste Upgrade, einige neue Features und Verbesserungen bietet folgende Vorteile. PerformancePoint-Dienste bietet verbesserte Scorecards, neue Visualisierungen, wie etwa den Analysebaum und KPI-Detailbericht und weitere Diagrammtypen besser Zeitintelligenz-Filterfunktionen und verbesserte Eingabehilfen Compliance.
